# Security Policy

## Supported versions

Security updates are applied to the latest released version.

## Reporting a vulnerability

Please do **not** open a public issue for security reports.

Preferred:
- Use GitHub **Security Advisories** (private report)

If you cannot use advisories, email:
- `security@0xstack.dev` (if not configured yet, open an advisory and we’ll respond there)

## What to include

- A clear description of the issue
- Steps to reproduce (proof-of-concept if possible)
- Impact assessment (what an attacker gains)
- Any suggested fixes/mitigations
