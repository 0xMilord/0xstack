## P0 Bugs — Ship-Stopping

### 1. `requireActiveOrg()` always redirects (orgs)
**File:** `lib/orgs/active-org.ts` — `requireActiveOrg()` references `gate.org` but `orgsService_resolveActiveOrg` returns `{ ok: true, orgId }` not `{ ok: true, org }`. Always undefined, always redirects.
**Impact:** No page using this helper will ever render.
**Fix:** Change service to return org object, or change helper to use `gate.orgId`.

### 2. API key prefix extraction is wrong (security-api)
**File:** `lib/services/api-keys.service.ts` — `key.slice(0, 8)` extracts `"0xstack_"` not the actual 8-hex prefix.
**Impact:** API key verification NEVER works. All API-protected routes silently accept any key or reject all keys.
**Fix:** Extract prefix from position 8 (after `0xstack_`), 8 chars before `.`.

### 3. Dodo checkout has no org metadata (billing-dodo)
**File:** `app/api/v1/billing/checkout/route.ts` — dumb pass-through to `@dodopayments/nextjs` Checkout. No metadata attached.
**Impact:** Webhook reconciliation stores `org_id = null`. Billing page always shows "no subscription."
**Fix:** Create custom checkout handler that attaches `orgId` + `userId` metadata (like Stripe module does).

### 4. Dodo portal route has no auth (billing-dodo)
**File:** `app/api/v1/billing/portal/route.ts` — no session check, no customer lookup.
**Impact:** Anyone can access the billing portal without authentication.
**Fix:** Require session, look up customer ID, create portal session.

### 5. Billing status API has no authz (billing-dodo)
**File:** `app/api/v1/billing/status/route.ts` — only calls `guardApiRequest`. Any API key can query any org's billing.
**Impact:** Information leak — billing data for any org accessible with any valid API key.
**Fix:** Add `requireAuth()` + org membership check.

### 6. Welcome email is NEVER triggered (email-resend + auth-core)
**File:** `lib/auth/auth.ts` — calls `global.sendWelcomeEmail` which is never assigned. Email-resend exports it but never assigns to global.
**Impact:** Welcome emails never send.
**Fix:** Wire `sendWelcomeEmail` into `databaseHooks.user.create.after` directly, not via global.

### 7. `lib/auth/server.ts` not owned by auth-core module
**File:** Generated by `run-init.ts`, not by auth-core module. Every other module imports from it.
**Impact:** If init is skipped or auth-core re-runs, this file may be missing or stale.
**Fix:** Move server.ts generation into auth-core module's activate().

### 8. PWA missing dependencies (pwa)
**Files:** `lib/pwa/push.ts` imports `web-push`, `lib/pwa/offline-storage.ts` imports `idb`. Neither package is installed.
**Impact:** Runtime crash on any PWA feature.
**Fix:** Add `web-push` and `idb` to deps when PWA enabled.

### 9. PWA icons 404 (pwa)
**Files:** `public/manifest.webmanifest`, `public/sw.js` reference `/icons/icon-192x192.png` etc. Files never generated.
**Impact:** PWA not installable on any browser.
**Fix:** Generate placeholder PNG icons during module activation.

### 10. Site-header hardcodes PWA imports (ui-foundation)
**File:** `components/layout/site-header.tsx` — unconditional `import { PwaUpdateBanner }` etc.
**Impact:** When PWA module is disabled, files are removed → TypeScript compilation FAILS.
**Fix:** Use dynamic import or generate header differently based on module config.

### 11. `lucide-react` not in deps (ui-foundation)
**Files:** theme-toggle, PWA components import from `lucide-react`. Not in `deps.ts`.
**Impact:** Crash on any page using theme toggle.
**Fix:** Add to baseline deps.

### 12. Jobs reconcile has no secret validation (jobs)
**File:** `app/api/v1/jobs/reconcile/route.ts` — checks `if (!secret) await guardApiRequest(req)` but never validates the secret value.
**Impact:** Anyone can trigger reconciliation by sending any `x-job-secret` header.
**Fix:** Validate against known `JOB_SECRET` env var.

### 13. Sentry uses deprecated pattern (observability)
**Files:** `sentry.*.config.ts` — deprecated in Next.js 14+. Needs `instrumentation.ts` with `register()`.
**Impact:** Server-side errors not captured, no distributed tracing.
**Fix:** Generate `instrumentation.ts` instead.

### 14. `withSentryConfig` missing from next.config (observability)
**Impact:** Source maps not uploaded, production errors show minified stacks.
**Fix:** Wrap `next.config.ts` with `withSentryConfig`.

### 15. Reconciliation never sets period fields (billing-dodo)
**File:** `lib/services/billing.service.ts` — `currentPeriodEnd` and `cancelAtPeriodEnd` columns exist but never populated.
**Impact:** Billing page cannot show subscription period info.
**Fix:** Extract from webhook payload.

### 16. Webhook replay has no idempotency (webhook-ledger)
**File:** `lib/services/webhook-ledger.service.ts` — calls `reconcileBillingEvent` every time, no `replayed_at` column.
**Impact:** Clicking "Replay" twice double-reconciles.
**Fix:** Add `replayed_at` column, skip if already replayed.

### 17. Webhook replay only supports Dodo (webhook-ledger)
**File:** Hardcoded `if (input.provider === "dodo")`. Stripe returns `provider_not_supported`.
**Impact:** Stripe webhooks that fail to reconcile cannot be replayed.
**Fix:** Add Stripe reconciliation support.

### 18. assets-client.tsx syntax error (storage-gcs)
**File:** `src={'/api/v1/storage/sign-read?assetId=' + a.id'}` — trailing quote.
**Impact:** Build crash.
**Fix:** Fix string concatenation.

### 19. API key fallback bypasses access control (storage-gcs)
**File:** sign-read route — API key path calls `storageService_createSignedRead` with NO userId or orgId.
**Impact:** Any API key holder can read ANY asset.
**Fix:** Require `ownerUserId` or `orgId` in body and verify.

### 20. API key fallback accepts arbitrary orgId (storage-gcs)
**File:** sign-upload route — API key path accepts `orgId` from body with NO membership validation.
**Impact:** Any API key holder can upload to ANY org.
**Fix:** Validate org membership.

### 21. `verifyApiKey` uses wrong security model (security-api)
**File:** `lib/security/api.ts` — `env.API_KEY` global key bypasses DB lookup entirely.
**Impact:** Single global key with no org scoping, no audit trail.
**Fix:** Remove global key bypass or document it as emergency-only.

### 22. Sign-read API has no authz (billing-dodo — webhook)
**Impact:** Webhook handler uses `@dodopayments/nextjs` Webhooks which does its own verification, but the standalone `verifyDodoWebhook` is dead code.
**Fix:** Remove dead code or use it if replacing the handler.

### 23. `auth.ts` overwritten by email module (email-resend)
**File:** `patchAuthForEmail()` does full file replacement, not a merge.
**Impact:** Any customizations from auth-core are wiped.
**Fix:** Use AST-based modification or composition API.

### 24. No Sentry middleware or instrumentation (observability)
**Impact:** No automatic error capture, no request tagging, no user identification.
**Fix:** Add `instrumentation.ts` with Sentry SDK initialization.

### 25. No OpenTelemetry implementation (observability)
**File:** `ctx.modules.observability.otel` exists in types but zero implementation.
**Impact:** Phantom feature — users who enable OTEL get nothing.
**Fix:** Implement or remove from schema.

### 26. No Inngest or cron driver wiring (jobs)
**Impact:** `driver: "inngest"` and `driver: "cron-only"` generate identical code.
**Fix:** Generate Inngest client/route for inngest driver, `vercel.json` for cron-only.

### 27. No scheduling mechanism for jobs (jobs)
**Impact:** Reconcile endpoint exists but nothing calls it automatically.
**Fix:** Generate `vercel.json` cron config or Inngest schedule.

## P1 Bugs — Functional Gaps

### 1. No `app/app/layout.tsx` generated (ui-foundation)
AppShell component exists but no layout wraps `/app/*` routes. All app pages render inside marketing layout.

### 2. Settings page path mismatches (ui-foundation)
Links to `/app/billing` but modules generate `/app/app/billing`. Same for assets, api-keys, pwa, webhooks.

### 3. SEO pages import from SEO module when disabled (ui-foundation)
about/contact/terms/privacy import `getPageMetadata` from `@/lib/seo/metadata`. When SEO is disabled, files are removed → build crash.

### 4. Blog index shows draft posts (seo+blog)
`listPosts()` returns ALL posts. Index doesn't filter `!p.published`. Drafts are publicly visible.

### 5. RSS feed includes draft posts (seo+blog)
Same issue — no published filter on RSS feed.

### 6. No `generateMetadata` on blog index (seo+blog)
Blog index has no title, description, OG tags, or canonical URL.

### 7. No blog OG image font (seo+blog)
`ImageResponse` renders without loading any font — ugly system fallback.

### 8. `softwareApplicationJsonLd` has fake rating (seo+blog)
Hardcoded `"5.0"` rating with `"1"` review — Google may flag as manipulative.

### 9. `articleJsonLd` helper never used (seo+blog)
Blog post page writes raw inline JSON-LD instead of using the helper. Duplicated, inconsistent.

### 10. No role-based authz for API key actions (security-api)
Any org member (including "member" role) can create and revoke API keys.

### 11. No role-based authz for destructive storage actions (storage-gcs)
Any org member can delete assets. No `assertRoleAtLeast` checks.

### 12. No role-based authz for billing checkout (billing-dodo)
Any org member can create subscriptions. No owner/admin gate.

### 13. No org membership check on API key loader (security-api)
Loader trusts "workspace layout guard" which may not exist. Read operations should verify membership.

### 14. Storage list API has no org membership validation (storage-gcs)
`GET /api/v1/storage/assets` reads active org cookie but doesn't call `assertMember`.

### 15. Billing status API has no org scoping (billing-dodo)
Even with auth, no check that caller is a member of the requested org.

### 16. No `withSentryConfig` in next.config (observability)
Source maps not uploaded to Sentry.

### 17. Email send functions have no error handling (email-resend)
If Resend is down, auth flows crash (password reset, verification).

### 18. No bounce/complaint handling (email-resend)
No webhook endpoint for Resend bounce events. Damages sender reputation.

### 19. No VAPID key generation helper (pwa)
Users must manually run `npx web-push generate-vapid-keys`. No CLI command or script.

### 20. PWA components orphaned (pwa)
`PwaUpdateBanner` and `PwaInstallButton` generated but never imported in any layout or page.

### 21. No navigation link to PWA settings (pwa)
No sidebar/navbar/menu links to `/app/pwa`. Undiscoverable.

### 22. Jobs reconcile hardcoded to "dodo" only (jobs)
If billing is Stripe, reconcile does nothing.

### 23. Webhook-ledger `validate()` and `sync()` are no-ops (webhook-ledger)
Module cannot detect drift (missing schema, env vars, etc.).

### 24. No mutation keys for webhook-ledger (webhook-ledger)
`lib/mutation-keys/webhook-ledger.keys.ts` never generated. Tests stub it manually.

### 25. No mutation keys for API keys (security-api)
`lib/mutation-keys/api-keys.keys.ts` never generated.

### 26. No v1 REST API for API key management (security-api)
No `app/api/v1/security/api-keys/route.ts`. Only server actions exist.

### 27. Missing security headers files (security-api)
`lib/security/request-id.ts`, `lib/security/headers.ts`, `lib/security/csp.ts` never generated despite being referenced in docs.

### 28. No pagination on storage list (storage-gcs)
Hardcoded `.limit(200)` for org assets, no limit for user assets.

### 29. No org invite/onboarding flow (orgs)
No invite generation, no email templates, no accept-invite page.

### 30. No org deletion or rename (orgs)
Only `insertOrg` and `getOrgById`. No update or delete.

## P2 Issues — Quality / Polish

### 1. GCS provider instantiates new Storage() per call (storage-gcs)
Should be module-level singleton.

### 2. Webhook upsert uses raw SQL with JSON.stringify (webhook-ledger)
Bypasses Drizzle parameterization. Use `.values()` API.

### 3. No `revalidate.webhookLedger()` on webhook ingestion (webhook-ledger)
Cache staleness — new events not reflected for up to 10 minutes.

### 4. Webhooks page has no filtering/pagination/feedback (webhook-ledger)
Hardcodes `{ limit: 50 }`, no provider filter, no date display, no replay feedback.

### 5. Webhook event schema not normalized across providers (webhook-ledger)
`payload_json` stores raw provider payload. `pick()` helper guesses field names.

### 6. No `status` or `processed_at` on webhook_events (webhook-ledger)
Cannot track processing state or failed events.

### 7. No `replayed_at` / `replay_count` columns (webhook-ledger)
No way to prevent double-reconciliation.

### 8. `verifyDodoWebhook` is dead code (billing-dodo)
Defined but never imported. `@dodopayments/nextjs` Webhooks handler does its own verification.

### 9. Client hook `use-billing.client.ts` is orphaned (billing-dodo)
Hook exists but no generated page uses it.

### 10. No plan change/upgrade/downgrade logic (billing-dodo)
Only "Start subscription" action exists.

### 11. No subscription status lifecycle UI (billing-dodo)
Raw status displayed. No handling for past_due, canceled, trialing, cancel_at_period_end.

### 12. `schema-dts` installed but never imported (seo+blog)
Only provides TypeScript types; code uses `as const` instead.

### 13. No cache headers on sitemap/robots (seo+blog)
Regenerated on every request.

### 14. OG image `?icon` param is dead code (seo+blog)
Read but never used in rendering.

### 15. OG image uses external font URL with no fallback (seo+blog)
Third-party URL failure = broken OG images.

### 16. `extractHeadings` regex is naive (seo+blog)
Matches headings inside code blocks.

### 17. No `generateStaticParams` for blog posts (seo+blog)
Dynamic rendering only — wasteful for static content.

### 18. `rehype-slug`/`rehype-autolink-headings` installed but not used (seo+blog)
TOC uses naive regex instead of actual heading IDs.

### 19. No author frontmatter field (seo+blog)
Frontmatter schema doesn't validate `author`. Post page hardcodes app name.

### 20. Blog index has no pagination (seo+blog)
All posts rendered at once.

### 21. Reading progress `<script>` in Server Component (seo+blog)
Bypasses React's rendering model.

### 22. No content revalidation mechanism (seo+blog)
No filesystem watcher or admin endpoint to trigger `revalidate.posts()`.

### 23. No draft/scheduled publishing workflow (seo+blog)
Only `published` boolean exists.

### 24. `PwaClient` can't show subscription status (pwa)
Receives only `vapidPublicKey` as prop. No `subscriptions` passed.

### 25. No unsubscribe capability in PwaClient (pwa)
Only server-rendered form can unsubscribe.

### 26. Offline storage is dead code (pwa)
`savePendingRequest` never called. No sync/replay mechanism.

### 27. No rate limiting on push send (pwa)
Unlimited POSTs to `/api/v1/pwa/push/send`.

### 28. Subscription upsert not atomic (pwa)
DELETE then INSERT without transaction.

### 29. PWA page shows raw endpoint URLs (pwa)
Not human-identifiable. No browser name or date.

### 30. `patchRootLayoutForPwa` may produce malformed JSX (pwa)
Injects `<head>` directly after `<html>` tag.

### 31. `SW_VERSION` hardcoded to "1.0.0" (pwa)
No auto-increment mechanism.

### 32. No `background_sync` handler in SW (pwa)
No sync event listener for offline queue.

### 33. No audit trail for API key lifecycle (security-api)
No `lastUsedAt`, `createdBy`, `revokedBy`, `updatedAt` columns.

### 34. In-memory rate limiter not shared across instances (security-api)
Serverless deployments have ineffective rate limiting.

### 35. `revokeApiKey` uses raw SQL (security-api)
Bypasses Drizzle column mapping.

### 36. `findActiveApiKeysByPrefix` hardcoded `.limit(5)` (security-api)
Not parameterized or documented.

### 37. Loader returns `[] as any[]` when no org (security-api)
Silent failure instead of explicit error.

### 38. `isMember` imported from repo instead of service (storage-gcs)
Violates CQRS contract.

### 39. No upload completion webhook/callback (storage-gcs)
No server notification when browser PUT to GCS completes.

### 40. No cleanup of orphaned GCS objects (storage-gcs)
Best-effort delete with `ignoreNotFound`. Orphans accumulate.

### 41. Image thumbnails hit API route per-card (storage-gcs)
N images = N POST requests to sign-read.

### 42. No file size limits enforced at GCS level (storage-gcs)
Signed URL has no size constraint.

### 43. `active-org.ts` cookie lacks `Secure` flag (orgs)
`HttpOnly; SameSite=Lax; Path=/` but no `Secure`.

### 44. No Zod validation on `setActiveOrg` (orgs)
Accepts `{ orgId: string }` raw. Should validate UUID format.

### 45. `assertRoleAtLeast` service function never called (orgs)
Exists but no generated action or API route uses it.

### 46. No org-scoped API surface (orgs)
No `/api/v1/orgs/*` routes for external consumers.

### 47. No centralized "revalidate all org-scoped caches" helper (orgs)
Each action manually lists revalidate calls. Fragile.

5. **orgs → billing-dodo** — billing needs org-scoped checkout (broken)
6. **orgs → storage-gcs** — storage needs org-scoped uploads (broken on API key path)
